# Spec: Kilroy Autonomous Daily Loop

Date: 2026-07-21. Decided via grilling session (Mac); execution target is Jordan's
Windows work PC (enterprise Claude account, Nova MCPs, corp network). Scaffolding is
built on the Mac and delivered via git.

Depends on: the connector-consolidation plan (approved 2026-07-20, decisions recorded in
`Knowledge/Lessons/2026-07-20-connector-consolidation-planning.md`). That change executes
FIRST — it retires AMR Hub / Master Tracker / `arriving-amr-progress`, guts
`fleet-commissioning-handoff` to Overmind-only, and moves Planner to Nova's `planner` MCP.
Everything below builds on the post-consolidation shape of the repo.

## Goal

Kilroy runs Jordan's workday loop autonomously on the Windows work PC, Monday–Thursday
(Jordan does not work Fridays). Four scheduled headless runs per day prepare everything;
Jordan approves the one external write (Confluence) interactively. Every run is journaled;
a weekly reflection turns journals into improvement drafts. The system gets measurably
smarter without ever silently editing itself.

## Schedule (Windows Task Scheduler, headless `claude -p`, Mon–Thu)

| Time  | Run          | Contents |
|-------|--------------|----------|
| 07:00 | Morning prep | Mentions/tags from mail + Teams DMs + configured group chats; Loop + Planner assignments; Confluence status draft; prioritized day checklist. Done by 08:30. |
| 08:30 | Retry        | Task Scheduler retry of the 07:00 run, once, only on failure (brief lands ~08:45 those days). |
| 11:00 | Delta        | New mentions, task movement since morning. |
| 13:00 | Delta        | Same pattern. |
| 16:00 | Close-out    | Carry-overs; Thursday adds the weekly reflection. |

Each run: headless `claude -p "/run-daily-workflow --autonomous --phase=<phase>"` from the
repo root, output logged, run journal appended, status marker written.

## Day checklist (the daily plan)

A checkbox checklist, generated by the morning run, ordered heavy-first:

- **Tier 1** — assigned tasks from Teams + Loop
- **Tier 2** — AMR repairs + commissioning work
- **Tier 3** — assigned tasks with no due date
- **Tier 4** — mentions needing a reply
- **Tier 5** — FYI / light admin (end of day)

Ties within a tier break by due date, then age. Tier rules are static but tunable — the
weekly reflection proposes tier-rule changes based on what Jordan actually reordered.

## Outputs

- Repo `Projects/daily/` is authoritative (git audit trail): brief, checklist, deltas,
  close-out, run journals under `Projects/daily/runs/`.
- Mirrored copy into the Obsidian vault (added via `--add-dir`; path from
  `OBSIDIAN_VAULT_PATH` in `.env`, never hardcoded). Vault copy is the one Jordan reads
  and annotates.

## Approval gate (non-negotiable, carried over)

Headless runs NEVER post to Confluence. Draft-then-approve stands exactly as written in
`Skills/confluence-daily-status/SKILL.md`. Structural backing: playwright/confluence
tools stay denied in the headless permission context even after the scoped allowlist is
built. First interactive session of the day surfaces the draft for approval; posting
happens only there, headed browser, never headless.

## Login hook (SessionStart)

Extends `.claude/hooks/session-start.sh`. On first interactive session of a workday:

1. Read today's run status markers.
2. All green → present brief: mentions digest, day checklist, Confluence draft awaiting
   approval.
3. Partial → present brief plus explicit per-source failure list (fail loud, per-source
   partials — one dead source never kills the brief, and never silently disappears).
4. Failed/absent → "morning run failed / never ran — run `/run-daily-workflow` now?"

## Failure handling

- One automatic retry at 08:30 (Task Scheduler setting), then fail loud at login.
- Per-source result recorded in the run status marker
  (`Projects/daily/runs/YYYY-MM-DD-<phase>.json`: `{ "status": "ok|partial|failed",
  "sources": { "<source>": { "status": "ok|failed", "detail": "<string>" } },
  "startedAt": "<ISO 8601>", "finishedAt": "<ISO 8601>" }`). Consistent with the
  repo-wide fail-loud rule.

## Learning loop

- Every scheduled run appends a structured journal entry: sources hit, per-source status,
  items surfaced, timing, anomalies.
- Thursday 16:00 run adds a reflection step: read the week's journals + lessons, propose
  concrete improvements (skill text tweaks, tier-rule changes, noise filters) as DRAFTS
  in `Knowledge/Lessons/` for Jordan's approval. Never edits skills or CLAUDE.md itself —
  the "no silent edits" anti-pattern applies with extra force to unattended runs.
- Approved drafts land through the normal skill-creator edit process.

## Loop (Microsoft Loop) access

New data source. Dual-path, resolved on first Windows run:
1. Nova `loop` MCP if present (Jordan expects it is).
2. Else Loop-assigned tasks via the `planner` MCP (Loop task lists sync to Planner/To Do).

Marked UNVERIFIED in the skill until the first Windows run confirms which path is real.
If neither works: fail loud, never silently skip Loop.

## Permissions

Empirical, not guessed: the first supervised run on the work PC captures every permission
prompt; the scoped allowlist in `.claude/settings.json` is built from that observed list.
Regardless of what the capture shows, playwright/confluence tools remain denied for the
headless context (see Approval gate).

## Bootstrap (first run on work PC — no 07:00 run that day)

Jordan clones the repo into the enterprise Claude account and logs in ~10:00–10:30. The
SessionStart hook detects first-run state (no `.env`, or a `bootstrap` marker) and walks a
setup checklist:

1. Ask Jordan for the specific Teams group chats to summarize in the morning brief →
   saved to config (`.env` / `Knowledge/Personal/`), not hardcoded.
2. Ask for the Obsidian vault path → `OBSIDIAN_VAULT_PATH`, register `--add-dir`.
3. Verify Loop path (Nova MCP vs Planner sync) and update the skill's UNVERIFIED marker.
4. Run the full morning prep on demand, supervised — this is also the permission-capture
   run. Deliver brief + checklist + Confluence draft immediately after.
5. Build the scoped allowlist from captured prompts.
6. Register the four Task Scheduler jobs (scripts provided in `scripts/windows/`).
7. Outstanding confluence-daily-status setup: dedicated Edge profile + one-time Confluence
   SSO sign-in, fill `.env`, restart Claude Code, `kilroy check`.

Scheduled cadence starts the following workday.

## Definition of done ("everything green")

1. `cd mcp-server && npm test` passes (post-consolidation suite).
2. `verify-fixtures` — all documented pairings pass.
3. `kilroy check` green on the work PC (post-consolidation scope).
4. Supervised bootstrap run completes; permission list captured; allowlist built.
5. One full unattended day observed: all four runs produce journals + repo/vault outputs.
6. One approved Confluence post lands correctly (right day row, nothing else moved, no
   watcher notification).
7. Loop path verified and skill updated.
8. Day checklist generates with correct tier ordering from real data.
9. First Thursday reflection produces an improvement draft.

Items 5–9 need real workdays; done lands ~1 week after Windows bootstrap.

## Out of scope

- Auto-posting to Confluence in any form (including timeout auto-post — explicitly
  rejected, again).
- Autonomous self-editing of skills or CLAUDE.md.
- Friday runs.
- Full Obsidian vault integration (daily-note templates, backlink maintenance) — future
  skill if the need repeats, per the anti-sprawl rule.
