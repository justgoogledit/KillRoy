# Inbox

Ad-hoc, non-AMR items Jordan drops in any time via "kilroy, add \<item\> to my list" or
"remind me to \<item\>". This is a scratch capture surface, not a deliverable -- Kilroy reads
`## Unconsumed` at the next morning brief (or midday pulse), folds every line into that day's
action list tagged `(jordan-request)`, and clears it from here. An item lives in exactly one
place at a time: either this file or a day file, never both.

See [[Skills/run-daily-workflow/SKILL|run-daily-workflow]] step 0 for the capture step and step 2
for the drain step.

Don't hand-edit a consumed item back in. If something needs to resurface, add it again --
that gives it a fresh timestamp and an honest trail of when it was actually re-raised.
