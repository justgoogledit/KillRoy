# Daily

Daily workflow plans land here. Filename pattern: `<YYYY-MM-DD>-daily.md`.

Produced by [[Skills/run-daily-workflow/SKILL|run-daily-workflow]]. Unlike handoffs and progress boards, the day file is a living document -- updated at the morning brief, the midday check, and the end-of-day close-out. One file per day; the close-out is the last write.

`inbox.md` also lives here -- not a dated snapshot but a single running scratch file. Jordan drops ad-hoc, non-AMR items into it any time ("add X to my list"); the next morning brief drains it into that day's action list and clears it. See [[Skills/run-daily-workflow/SKILL|run-daily-workflow]] step 0.
