# Lessons

Output of `session-recap`. One file per session worth recording.

Filename: `YYYY-MM-DD-<slug>.md`

See `Skills/session-recap/SKILL.md` for the template Claude uses to write these.

The vault-root `log.md` is an append-only index of every lesson, so future sessions can find recent learnings without reading every file here.

## Anti-pattern

Don't write a lesson if nothing reusable happened. An empty `Lessons/` folder after a session is fine. A folder full of "user said hi" entries is noise that pollutes future context.
