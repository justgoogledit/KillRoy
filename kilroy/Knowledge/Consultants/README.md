# Consultants

Distilled expertise from people you trust on specific topics. Think of these as voices you can summon for a second opinion.

Each consultant gets a file: `<name>.md`. Capture their core principles, decision heuristics, and how they think — enough that Claude can simulate their perspective when you say "what would <name> do here?"

## What makes a good consultant entry

- **Core principles** — 3-7 ideas they're known for
- **Decision heuristics** — how they cut through ambiguity
- **Things they push back on** — pet peeves, common pitfalls they warn against
- **A few signature quotes** — verbatim, sourced

## Examples of people worth consultant-ifying

- A senior engineer whose code reviews you trust
- An author whose book changed how you think about a topic
- A speaker whose talk you keep coming back to
- A colleague whose taste you'd want to borrow

The `consultant-query` skill (not included in the starter pack — build it when you have 2+ consultants worth querying) makes this routine.
